# Security Policy for Systemly

## Supported Versions

This section outlines which versions of **Systemly** are currently supported with security updates.

| Version       | Supported          |
| ------------- | ----------------- |
| 1.0.x – 2.0.x | :x: Not Supported |
| 2.1.x – 3.0.x | :white_check_mark: Supported |
| 3.1.x – 4.0.x | :white_check_mark: Supported |
| 4.1.x – Current | :white_check_mark: Supported |

> ⚠️ We strongly recommend using the latest supported version to ensure you receive the latest security fixes.

---

## Reporting a Vulnerability

Systemly takes security seriously. If you discover a vulnerability, please follow the steps below:

### Contact

- Email: **security@systemly.io**  


### Required Information

When reporting a vulnerability, please include:

- Description of the vulnerability  
- Steps to reproduce  
- Affected version(s)  
- Any suggested fixes (optional)

### Response & Timeline

- We will acknowledge your report within **48 hours**.  
- Security updates or patches will be prioritized based on severity.  
- You will receive regular updates until the issue is resolved.

### Disclosure Policy

- Please allow Systemly sufficient time to address the vulnerability before public disclosure.  
- Coordinated disclosure ensures that users are protected while fixes are implemented.

### Recognition

- While Systemly currently does not offer a bounty program, responsible disclosure will always be acknowledged publicly if you wish.
